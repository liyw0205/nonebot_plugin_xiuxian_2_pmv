# 链接解析（流媒体）

娱乐模块自带原生链接解析，**不依赖**外部 core 下载。把分享链接发给机器人即可（也可用指令）。

## 指令

| 指令 | 别名 | 说明 |
|:-----|:-----|:-----|
| `链接解析 <链接>` | `视频解析` / `解析视频` / `解析链接` / `流媒体解析` | 解析并发送卡片 + 视频/图集 |
| 直接发链接 | — | 消息内含支持域名时自动识别（受去重与群限流约束） |

帮助入口：`娱乐帮助`（可翻页）。

## 支持平台

| 平台 | 示例域名 | 备注 |
|:-----|:---------|:-----|
| B 站 | `bilibili.com` / `b23.tv` | 视频多清晰度；动态/opus 图文图集；视频会尽量本地下载后发送 |
| 抖音 | `douyin.com` / `iesdouyin.com` | 视频 / 图集 |
| 快手 | `kuaishou.com` | 单作品主视频 + 封面；图集走 atlas 全量图 |
| 小红书 | `xiaohongshu.com` / `xhslink.com` | 图文为主；部分环境需代理或带 `xsec_token` |
| 微博 | `weibo.com` / `t.cn` | 图 / 视频 |
| 小黑盒 | `xiaoheihe.cn` | 帖子/分享；若接口要求人机验证会明确提示 |
| X / Twitter | `x.com` / `twitter.com` | 建议开启自定义代理 |
| TikTok | `tiktok.com` | 建议代理；可能 429 / 登录墙 |
| Instagram | `instagram.com` | 建议代理；环境限制较多 |
| YouTube | `youtube.com` / `youtu.be` | 建议代理；走 yt-dlp，受出口与地区影响 |

## 发送表现

1. **信息卡片**：标题、作者头像、发布时间、平台图标（横屏 logo 等比缩放，不拉伸）。
2. **媒体**：优先发**最高清晰度**；单文件超过约 **20MB** 时自动降档到更小清晰度。
3. **原始链接**：卡片/文案尽量展示用户发来的**短链**，而不是展开后的落地页。
4. **群限流**：QQ 被动回复次数紧张时优先视频，并识别 `40034128` 等限流错误，避免连发刷屏。

## 网络代理

在 `xiuxian_config.py` 或 **Web 面板 → 配置管理 → 网络代理**：

| 配置 | 说明 |
|:-----|:-----|
| `custom_proxy_enabled` | 是否启用自定义代理 |
| `custom_proxy` | 代理地址（如 `socks5h://user:pass@host:port`） |

默认策略（实现以源码为准）：

- **走代理**：X / TikTok / Instagram / YouTube（及小红书直连失败时的回退场景）
- **国内站直连**：B 站、抖音、快手、微博、小黑盒等

依赖：`PySocks`、`yt-dlp`、`curl_cffi`（见根目录 `requirements.txt`）。

## 常见问题

| 现象 | 可能原因 | 建议 |
|:-----|:---------|:-----|
| 视频发送失败 / 850026 | 平台直链对 QQ 不可用 | 已尽量本地下载再发；检查磁盘与网络 |
| 小黑盒 `show_captcha` | 接口风控 | 换网络或稍后；非功能未实现 |
| YouTube / TikTok / IG 失败 | 429、登录墙、出口限制 | 配置可用代理；接受平台限制 |
| 小红书打不开 | 直连或 token | 开代理；使用完整分享链 |
| 图集只出封面 | 平台结构变化 | 更新插件后再试；看日志中的解析错误 |
