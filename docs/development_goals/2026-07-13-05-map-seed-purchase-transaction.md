# 目标 05：地图种子购买原子化

**状态：已完成（2026-07-13）。**

## 范围

将地图种子商店的灵石扣款与背包入库收敛到可幂等事务。

## 风险与依据

当前 `购买种子` 连续执行扣灵石和发放种子，后一步失败或重复命令会导致资产不一致。

## 验收

- 事务内复核玩家、灵石、背包容量和种子参数；
- 基于消息 ID 去重；
- 失败时不扣灵石且不入库；
- 测试成功、重复、余额不足、容量不足和写入失败。

## 实现记录

- `SeedPurchaseService` 在单个 SQLite 事务内复核灵石、背包容量并提交扣款、种子入库和幂等操作记录；
- 命令按消息 ID 生成操作键，重复投递复用第一次购买结果；
- 已验证成功、余额不足、背包已满、陈旧状态、重复提交和操作记录写入失败回滚。
