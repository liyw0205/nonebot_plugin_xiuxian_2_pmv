# 目标 04：悬赏令结算原子化

**状态：已完成（2026-07-13）。**

## 范围

将悬赏完成标记、修为奖励和额外物品奖励作为一个结算边界。

## 风险与依据

悬赏结算分别更新修为、工作状态与背包；中断或重试会导致重复奖励或任务状态未完成。

## 验收

- 随机结果在进入服务前固定；
- 事务内复核悬赏仍可结算并写入完成状态；
- 修为、物品和完成状态同成败；
- 覆盖重复事件、修为上限、无物品、写入失败和回滚。

## 实现记录

- `WorkSettlementService` 在同一 SQLite 事务内更新修为、悬赏状态、背包奖励和幂等操作记录；
- 随机结算与大成功倍率在服务调用前固定，服务按实时修为上限截断实际收益；
- 已验证成功、上限截断、背包已满、重复提交、冲突重试和操作记录写入失败回滚。
