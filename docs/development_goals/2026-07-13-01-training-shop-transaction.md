# 目标 01：历练商店兑换原子化

**状态：已完成（2026-07-13）。**

## 范围

将 `历练兑换` 的成就点扣减、周限购计数与背包入库合并为单个 SQLite 事务。

## 风险与依据

当前命令依次写入 `training` 和背包；任一后续写入失败会留下已扣成就点或已占用限购但未发货的状态。重复 Bot 事件也会重复扣点。

## 验收

- 使用消息 ID 建立操作幂等键；
- 余额、周限购、背包容量和玩家存在性在事务内复核；
- 任一失败回滚成就点、限购和背包；
- 重试不重复扣点或发货；
- 增加临时 SQLite 行为测试。

## 实现记录

- `TrainingPurchaseService` 使用跨库事务提交成就点、周限购和背包；
- 相同操作 ID 复用首个结果，冲突载荷和陈旧状态均拒绝；
- 已验证成功、限购、余额不足、容量不足、陈旧状态、重复提交和写入失败回滚。
