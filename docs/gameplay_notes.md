# 常用玩法说明（玩家 / 服主）

完整指令以群内 **`修仙帮助` / `娱乐帮助` / `修仙手册`** 为准。本文只补充近期行为与易踩坑点。

## 悬赏令

| 指令（示例） | 说明 |
|:-------------|:-----|
| `悬赏令刷新` | 刷新可选任务（有次数限制） |
| `悬赏令确认刷新` | 确认本次刷新 |
| `悬赏令接取 1/2/3` | 接取列表中第几条 |
| `悬赏令结算` | 时间到后结算 |

注意：

1. **先刷新出列表**再接取；没有 offer 时会提示先刷新。  
2. 同一 openid 若在库里有**重复角色行**，旧逻辑可能误报「状态或刷新次数已变化」；当前版本刷新会兼容重复行。长期仍建议在库里合并脏账号。  
3. 每日次数由定时任务重置（Web「定时任务」里可见「悬赏令次数重置」）。

## 秘境 · 秘藏令

- 普通进入：同一轮秘境不可重复进。  
- **秘藏令**（ticket）：可在已参加过本轮的情况下**额外再进**（仍要求角色当前为空闲等业务条件）。  
- 以群内秘境相关帮助与道具说明为准。

## 探索结算

- 指令：`探索结算`（地图探索结束后）。  
- 若出现「探索结算数据异常」：多为历史脏快照（如 `reward_plan` 被写成字面量 `None`）。当前版本会清空脏数据并按探索状态重算，一般可直接再结或已自动可结。  
- 结算成功后探索状态会清零，无需对同一笔重复结算。

## 双修 / 高境界数值

- 高境界经验值极大时，旧版用 SQL `CAST(exp)` 精确比对可能**假失败**（看起来没加经验）。  
- 当前双修、道侣/师徒相关返修与传功已改为更稳妥的整数比对；若仍异常请看日志并反馈 user_id。  
- 带按钮的失败提示会带上按钮参数，避免界面显示 `None`。

## 称号自动检查

- 成就/称号自动检查依赖玩家数据管理器；旧版缺导入会导致日志 `player_data_manager is not defined`。  
- 已修复后自动检查不再刷该错误；玩家侧一般无感。

## 定时任务（服主）

日常重置、备份、鬼市、傀儡收取、世界事件等见 Web 面板 **定时任务** 页，或 [web_panel.md](web_panel.md)。  
玩家侧只需知道：次数类玩法会在配置的时间点重置。
